Project Overview

The Paterson AI Street Cleanliness System uses AI-powered video analysis from garbage truck dashcams to automatically detect and document street cleanliness violations across Paterson, NJ. The system processes dashcam footage, aligns video frames with GPS coordinates, scores street cleanliness using the Keep America Beautiful (KAB) methodology, and auto-generates warning letters to nearby property owners.

Known Limitations

  1. Resolution: 640×360 dashcam footage limits small-item detection. Higher resolution (960×540+) significantly improves accuracy.
  2. Boundary cases: Snow patches and small litter are sometimes indistinguishable even to human reviewers (3/39 frames). System flags these as MEDIUM for human review.
  3. Special locations: Garbage collection facilities show as flagged. City should provide facility location list for filtering.
  4. Owner data: Warning letters use placeholder names. City plugs in owner database locally in production.

KAB Scoring Methodology

Each video frame is scored on 5 dimensions using the Keep America Beautiful Community Appearance Index (1–4 scale, 1 = best). The system also flags illegal dumping as a binary detection.

Dimension1 (Best)234 (Worst)
LitterMinimalSome visibleCatches eye frequentlyExtremely littered
GraffitiNoneFew tagsMultiple tagsContinuous
Abandoned VehiclesNone1 vehicle2–3 vehicles4+ vehicles
Outside StorageNone1–4 itemsFills small shedFills garage
Illegal SignsNone1–5 signs6–9 signs10+ signs

Confidence: clean → LOW → MEDIUM → HIGH (HIGH/MEDIUM trigger warning letters)

Two-Tier AI Architecture

Single-Step Pipeline (Production)

GPT-4o-mini screens all frames (~$0.0003/frame), then GPT-4o scores flagged frames (~$0.004/frame) with temperature=0.

Validated: 89% accuracy, 0 false positives at HIGH.

Two-Step Pipeline (Prototype)

Step 1: GPT-4o describes scene (no scoring). Step 2: GPT-4o matches description to KAB rules (no image).

Benefit: Transparency & auditability. Caught 1 extra issue single-step missed.

Deployment Guide

Requirements

  • Python 3.12+ (python.org)
  • ffmpeg (ffmpeg.org)
  • OpenAI API account (platform.openai.com)
  • Samsara API token (city-owned, GPS read access)

Install Dependencies

pip install -r requirements.txt # requirements.txt contents: # openai, pandas, numpy, matplotlib, opencv-python, # requests, folium, openpyxl, pyyaml, pillow

Set API Keys

Create a .env file from .env.example:

OPENAI_API_KEY=sk-proj-your-key-here SAMSARA_API_TOKEN=samsara_api_your-token-here

Or set environment variables directly:

# Windows CMD set OPENAI_API_KEY=your-key set SAMSARA_API_TOKEN=your-token # PowerShell $env:OPENAI_API_KEY="your-key" $env:SAMSARA_API_TOKEN="your-token"

Configuration

Edit config.yaml to set the active vehicle, paths, and pipeline settings:

active_vehicle: "R75_jan23" # or "R21_mar02" pipeline: screening_model: "gpt-4o-mini" verification_model: "gpt-4o" temperature: 0 shotgun_count: 5 # properties per detection

Alternative: Two-Step Pipeline

For transparency and auditability, use the two-step pipeline instead of step 4:

python pipeline/two_step_analyze.py

This separates scene description (what the AI sees) from rule matching (how it scores), allowing human review of the reasoning chain.

Data Format

timestamp,latitude,longitude,speed,heading,address 2026-01-23T11:02:48Z,40.923696,-74.155494,0.0,0,"410 East 16th Street, Paterson, NJ, 07514" 2026-01-23T11:03:08Z,40.923691,-74.155495,0.335,65,"410 East 16th Street, Paterson, NJ, 07514"
  • timestamp: UTC ISO format with Z suffix
  • latitude/longitude: WGS84 coordinates
  • speed: Vehicle speed in mph (frames with speed < 1.0 mph are filtered out)
  • heading: Compass direction (0–360°), not present in all files
  • address: Reverse-geocoded street address from Samsara API
